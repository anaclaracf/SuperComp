Relatório Intermediário - Alinhamento de Sequencias de DNA¶

Super Computação¶

Aluna: Ana Clara Carneiro de Freitas

Este projeto foi desenvolvido com o intuito de testar e desenvolver diversas estratgias para realizar o alinhamento de sequncias de DNA, o qual consiste em comparar duas sequncias de bases para observar seu nvel de similaridade. Tal similaridade calculada por meio dos acertos (matchs), os quais contribuem com um sinal positivo nessa similaridade, e dos erros (gaps e mismatchs), que contribuem negativamente.

Dessa maneira, visando encontrar um alinhamento que maximize esse nvel de similaridade, foram implementadas trs estratgias, sendo elas uma Soluo Heurstica, uma Busca Local e, por fim, uma Busca Exaustiva.

def roda_com_entrada(ex, in_f):
    with open(in_f) as f:
        start = time.perf_counter()
        proc = subprocess.run([ex], input=f.read(), text=True, capture_output=True)
        end = time.perf_counter()

    return proc.stdout, end-start

Solução Heurística de Alinhamento Local de Smith-Waterman¶

A primeira estratgia implementada foi a heurstica, a qual realiza seu alinhamento de sequncias de DNA por meio do algoritmo smith-waterman. Esta, para encontrar um alinhamento local otimo, realiza o calculo da diagonal, deleo e insero de cada um dos elementos da matriz, para assim, descobrir o maior valor entre eles e encontrar o melhor alinhamento das sequencias.

Inicialmente, visando averiguar a performance do algoritmo em relao ao tamanho da entrada, foram criados diversos arquivos que possuem sequencias com tamanhos diferentes, desde muito pequenos muito grandes, para testar todos os tipos de cenrios.

Algoritmo de Busca Local para Alinhamento¶

O segundo algoritmo implementado foi o de busca local, que diferente do apresentado acima, conta com aleatoriedade. O fato da aleatoriedade ter sido implementada tras um aspecto positivo ao resultado do alinhamento, podendo transformar a soluo em tima.

Este tambm, em vez de realizar o alinhamento entre duas sequencias, faz entre uma sequencia e uma subsequencia, a qual selecionada por aleatoriedade. Alm disso, este foi rodado diversas vezes para que fosse possvel analisar de forma mais precisa os resultados e o comportamento em relao aos valores aleatrios k, tamnho mximo da subsequencia, e p, quantidade de tentativas de alinhamnetos entre as subsequencias para encontrar o melhor score.

Igual ao algoritmo anterior, foi feito uma mdia entre os tempos de arquivos com entradas do mesmo tamanho, para melhorar a preciso dos tempos.

Este algoritmo se difere dos outros ao realizar o alinhamento das sequencias por meio de suas subsequencias, o que permite gerar todas as solues viveis para o problema, elegindo assim uma soluo tima. A primeira busca exaustiva implementada nesse projeto contou com a estratgia do Smith-Waterman, explicada anteriormente, para calcular os alinhamentos, o qual permite a utilizao de subsequencias de tamanhos distintos.

Para esse tipo de busca, foi necessrio utilizar arquivos com entradas menores, devido a complexidade do cdigo, o que impossibilitou que grandes valores fossem rodados, o que ser explicado mais para frente no relatrio por meio do profiling.

Essa busca exaustiva, diferente da outra, utiliza a tcnica de truncamento para deixar as subsequencias com o mesmo tamanho, pegando o valor da menor como referncia. E, a partir desse truncamento, foi realizado o clculo do score de maneira simples, onde, caso as letras da subsequencia fossem iguais seria somado dois a esse e score e, caso contrrio, seria subtrado um.

Por ser uma implementao menos complexa, foi possvel utlizar entradas com tamanhos maiores do que a anterior.

Paralelismo com OpenMP¶

Diferente das outras quatro estratgias apresentadas acima, essa busca explorar a execuo em paralelo do modelo de forma multicore (CPU) com OpenMP.

Dessa forma, a estratgia foi utilizar o cdigo da busca exaustiva com o alinhamento feito por Smith-Waterman como modelo base e, a partir dele, identificar os possveis pontos de paralelizao e, com isso, aplicar o OpenMP nesses pontos. O ponto de maior estresse do cdigo, encontrado por meio de ferramentas de profiling, foi na chamada de funo responsvel por calcular os scores.

Inicialmente, visando averiguar a performance do algoritmo em relao ao tamanho da entrada, foram criados diversos arquivos que possuem sequencias com tamanhos diferentes, desde muito pequenos muito grandes, para testar todos os tipos de cenrios.

Paralelismo com GPU¶

Por fim, a ltima estratgia implementada foi a de paralelizao manycore em GPU, que, assim como a de multicore, contou com a exaustiva de Smith-Waterman como cdigo base para a implementao. Para realizar a paralelizao desse cdigo, houveram algumas mudanas as quais buscavam deixar o cdigo mais eficiente, como por exemplo o clculo do max score ser realizado na GPU e a implementao de OpenMP na seo responsvel por gerar as possveis combinaes entre as sub sequncias. Entretanto, mesmo com essas implementaes, essa estratgia continuou sendo uma das menos eficintes do projeto.

Diante disso, uma possvel otimizao do cdigo seria enviar todas as sub sequncias de A e B para a GPU para que l fosse implementado suas possveis combinaes juntamente com o calculo do max scores, diminuindo a parte sequencial do cdigo. Afinal, com uma vazo de dados maior, a GPU conseguiria performar de forma mais eficinte e, com isso, entregar melhores resultados.
